Cisco IOS Command Reference

Organised by category — click a tab to jump to that section. Mode abbreviations: Priv = privileged EXEC, Conf = global config, If = interface config.

CommandModeWhat it shows
show running-configPrivilegedThe current active configuration in RAM — most-used verification command
show startup-configPrivilegedThe saved configuration in NVRAM — what the device loads on next reboot
show versionPrivilegedIOS version, uptime, hardware model, memory, licence info
show ip interface briefPrivilegedOne-line status of every interface — IP, status (up/down), and protocol (up/down)
show interfacesPrivilegedDetailed stats for all interfaces — errors, bandwidth, duplex, MTU
show interfaces Gi0/0PrivilegedDetailed stats for a specific interface (replace Gi0/0 with your interface name)
show ip routePrivilegedFull routing table — all known routes with source codes (C=connected, S=static, O=OSPF)
show arpPrivilegedARP cache — IP-to-MAC mappings the device has learned
show mac address-tablePrivilegedSwitch MAC table — which MAC addresses are learned on which ports and VLANs
show vlan briefPrivilegedList of all VLANs and which ports are assigned to each
show interfaces trunkPrivilegedActive trunk ports, their encapsulation, native VLAN, and VLANs allowed/active
show spanning-treePrivilegedSTP topology — root bridge, port roles (Root/Designated/Blocked), port states
show ip ospf neighborPrivilegedOSPF adjacency table — neighbour IDs, states (should be FULL), dead timer
show ip ospf databasePrivilegedOSPF Link-State Database — all LSAs the router has received
show access-listsPrivilegedAll configured ACLs and their match counters
show ip nat translationsPrivilegedActive NAT translation table entries
CommandModeWhat it does
enableUser EXECEnter privileged EXEC mode — required before any configuration or verification commands
configure terminalPrivilegedEnter global configuration mode — all configuration changes start here
hostname nameGlobal ConfigSet the device's hostname — shown in the prompt and in show version
enable secret passwordGlobal ConfigSet an MD5-hashed password for privileged EXEC — always use secret instead of password
line console 0Global ConfigEnter console line config — used to set console login password
line vty 0 4Global ConfigEnter virtual terminal (SSH/Telnet) line config — 0 4 covers all 5 vty lines
password pass / loginGlobal ConfigSet a line password and enable login prompt on that line
service password-encryptionGlobal ConfigEncrypts all plaintext passwords in the config with weak Type 7 encoding
banner motd # message #Global ConfigSet a login warning banner — the # is the delimiter, any character works
interface Gi0/0Global ConfigEnter interface configuration for a specific interface (Fa0/0, Gi0/0, etc.)
ip address X.X.X.X M.M.M.MInterfaceAssign an IP address and subnet mask to the interface
no shutdownInterfaceBring the interface up — router interfaces are shutdown by default, this enables them
description textInterfaceAdd a text label to an interface — visible in show interfaces and running-config
copy running-config startup-configPrivilegedSave the configuration. Without this, all changes are lost on reboot. Shortcut: wr
reloadPrivilegedReboot the device — loads startup-config. Useful to test if config survives a reboot
erase startup-configPrivilegedWipe the saved config from NVRAM — device will boot with blank config after next reload
CommandModeWhat it does
vlan 10Global ConfigCreate VLAN 10 and enter VLAN config mode — the VLAN is saved to vlan.dat
name VLAN_NAMEVLAN ConfigAssign a descriptive name to the VLAN — best practice for readability
interface Fa0/1Global ConfigEnter interface config to assign it to a VLAN
switchport mode accessInterfaceSet the port to access mode — connects to a single end device, carries one VLAN only
switchport access vlan 10InterfaceAssign the access port to VLAN 10
switchport mode trunkInterfaceSet the port to trunk mode — carries traffic for multiple VLANs between switches
switchport trunk native vlan 99InterfaceChange the native VLAN from the default (VLAN 1) to VLAN 99 — security best practice
switchport trunk allowed vlan 10,20,99InterfaceRestrict which VLANs are allowed to traverse this trunk link
no vlan 10Global ConfigDelete VLAN 10 — ports assigned to it will go inactive
interface vlan 10Global ConfigCreate an SVI (Switched Virtual Interface) — used for inter-VLAN routing on L3 switches
ip routingGlobal ConfigEnable Layer 3 routing on a multilayer switch — required for inter-VLAN routing
CommandModeWhat it does
ip route network mask nexthopGlobal ConfigAdd a static route — e.g. ip route 10.0.0.0 255.255.255.0 192.168.1.1
ip route 0.0.0.0 0.0.0.0 nexthopGlobal ConfigDefault route — forward all unmatched traffic to this next hop (your ISP or gateway)
no ip route network mask nexthopGlobal ConfigRemove a static route from the routing table
show ip route staticPrivilegedShow only static routes in the routing table
show ip route connectedPrivilegedShow only directly connected networks (interfaces that are up with an IP)
ping X.X.X.XPrivilegedTest reachability to an IP address using ICMP echo — 5 packets by default
traceroute X.X.X.XPrivilegedTrace the hop-by-hop path packets take to a destination — invaluable for troubleshooting
ip default-gateway X.X.X.XGlobal ConfigSet default gateway on a switch (not a router) — used when the switch needs management access
CommandModeWhat it does
router ospf 1Global ConfigEnable OSPF process 1 and enter OSPF config mode — process ID is locally significant only
router-id 1.1.1.1OSPF ConfigManually set the OSPF Router ID — strongly recommended over letting IOS auto-select one
network X.X.X.X W.W.W.W area 0OSPF ConfigAdvertise networks into OSPF — W.W.W.W is the wildcard mask (inverse of subnet mask)
passive-interface Gi0/0OSPF ConfigStop OSPF hellos on an interface (e.g. LAN-facing ports) but still advertise the network
passive-interface defaultOSPF ConfigMake all interfaces passive by default — then selectively enable with no passive-interface
default-information originateOSPF ConfigAdvertise the default route (0.0.0.0/0) into OSPF so other routers learn how to reach the internet
auto-cost reference-bandwidth 1000OSPF ConfigAdjust the reference bandwidth to 1000 Mbps — ensures GigE and FastE have different costs
show ip ospfPrivilegedOSPF process info — Router ID, area, SPF count, timers
show ip ospf neighborPrivilegedNeighbour table — state must be FULL for correct operation (2WAY is ok for DROthers)
clear ip ospf processPrivilegedRestart the OSPF process — forces re-election of Router ID and DR/BDR
CommandModeWhat it does
access-list 10 permit X.X.X.X W.W.W.WGlobal ConfigStandard ACL (1–99) — permit traffic from matching source IPs. W.W.W.W = wildcard mask
access-list 100 permit tcp src wild dst wild eq 80Global ConfigExtended ACL (100–199) — permit TCP traffic from source to destination on port 80
access-list 100 deny ip any anyGlobal ConfigExplicit deny all — the implicit deny is invisible, adding this makes it visible in show access-lists
ip access-list extended NAMEGlobal ConfigCreate a named extended ACL — easier to read and allows deleting individual lines by sequence number
interface Gi0/0 → ip access-group NAME inInterfaceApply ACL to an interface in the inbound direction — filters packets arriving on this interface
ip access-group NAME outInterfaceApply ACL in the outbound direction — filters packets leaving this interface
show access-listsPrivilegedShow all ACLs with match counters — useful to verify which rules are hitting traffic
show ip interface Gi0/0PrivilegedConfirm which ACL is applied inbound/outbound on a specific interface
no access-list 10Global ConfigDelete an entire numbered ACL — careful: this removes all rules at once
CommandModeWhat it does
ip nat inside source list 1 interface Gi0/1 overloadGlobal ConfigPAT (overload) — translate all IPs matching ACL 1 to the router's outbound interface IP. "overload" = many-to-one
ip nat inside source static 10.0.0.10 203.0.113.10Global ConfigStatic NAT — permanently map one inside local IP to one inside global IP (e.g. for a web server)
ip nat inside source list 1 pool POOLGlobal ConfigDynamic NAT using a pool — translate IPs in ACL 1 to an address from the named pool
ip nat pool POOL start end netmask maskGlobal ConfigDefine a NAT pool of public IP addresses to use for dynamic NAT translations
interface Gi0/0 → ip nat insideInterfaceMark this interface as inside — faces the private network
interface Gi0/1 → ip nat outsideInterfaceMark this interface as outside — faces the public/internet side
show ip nat translationsPrivilegedShow the active NAT translation table — each active session has an entry here
clear ip nat translation *PrivilegedClear all dynamic NAT translations — useful when troubleshooting or testing changes
show ip nat statisticsPrivilegedNAT counters — total translations, hits, misses, inside/outside interfaces
Caution in production: Debug commands generate real-time output and can consume significant CPU on a live device. Always use undebug all to stop debugging when finished.

Simulator Tips & Shortcuts

Time-saving tricks for Cisco Packet Tracer and GNS3 that every student should know.

IOS command shortcutYou can abbreviate any command as long as it's unambiguous — en for enable, conf t for configure terminal, sh ip int br for show ip interface brief
? for context helpType ? at any point in IOS for a list of valid commands or options — e.g. show ? lists all show commands, show ip ? narrows it further
Tab completionPress Tab to auto-complete a command or keyword. Works with partial input — saves typing and reduces errors
Command historyUse ↑ / ↓ arrow keys to cycle through recent commands in IOS — same as a shell. Saves retyping long commands
no commandPrepend no to any IOS command to undo it — no shutdown enables an interface, no ip address removes the IP, no router ospf 1 removes OSPF
Packet Tracer: PDU captureIn Simulation mode, use Add Simple PDU (the envelope icon) to send a test packet and watch it traverse each device hop-by-hop with layer details
Save early, save oftenBoth Packet Tracer and GNS3 projects are separate from the device configs. Save the project file and run copy run start on each device — they are two different saves
GNS3: Wireshark captureRight-click any link in GNS3 → Start capture — Wireshark opens automatically and shows live packet captures on that link. Invaluable for understanding protocols
