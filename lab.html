Lab 02 Beginner

VLAN Segmentation & Trunking

Guest Wi-Fi traffic is leaking into the corporate LAN — users can reach servers they should never see. You'll fix the switch configuration by creating VLANs, assigning ports, and setting up a trunk link so the two groups are completely isolated.

90–120 min estimated
Packet Tracer
Networking Track

The Scenario

You're the only network engineer at a 50-person company. The office has two groups of users: Corporate employees who need access to internal file servers and sensitive systems, and Guest users — contractors and visitors who connect to a separate Wi-Fi SSID.

During a routine security check, you discover that guest laptops can ping — and in some cases browse — corporate file servers. Both groups are on the same flat Layer 2 network with no separation. Someone set it up years ago and nobody ever fixed it.

Your job: reconfigure the switches to put corporate and guest traffic on separate VLANs. When you're done, a guest device should be completely unable to communicate with any corporate device — not because of a firewall, but because the switch never even delivers the frames.

Before You Start

Cisco Packet Tracer installed This lab uses Cisco Packet Tracer — a free simulator from Cisco. If you haven't installed it yet, expand the Simulator Setup section above the lab steps for a full download and install guide. A 2960 switch in Packet Tracer runs the same IOS commands used here.
You understand what a switch does at Layer 2 A switch moves Ethernet frames between devices using MAC addresses. It does not route IP packets — that's a router's job. This lab lives entirely at Layer 2. If you're fuzzy on this, read up on OSI Layer 2 first.
You've used Cisco IOS CLI before You should know how to type enable to enter privileged mode, configure terminal to enter global config, and how to navigate between config modes. If not, complete Lab 01 first — it walks through the basics.
Basic IP addressing knowledge You need to know what a /24 subnet means and how to set a static IP address on a host. You won't be doing any subnetting math here — just assigning IPs from two pre-defined subnets (10.0 and 20.0).

Go Further

You've mastered the basics. These open-ended challenges have no guided steps — research, experiment, and figure it out. That's what real networking looks like.

Silver
Router-on-a-Stick
A manager needs access to both VLANs. Configure a router connected to SW1 with two subinterfaces (one per VLAN) so selective inter-VLAN routing is possible, without flattening the VLANs.
Look up encapsulation dot1Q on router subinterfaces. The switch port connecting to the router should be a trunk. Each subinterface needs its own IP address — that IP becomes the default gateway for that VLAN.
Gold
Port Security
Lock down all access ports so each one only permits a single MAC address. Set violation mode to restrict with logging. Then enable BPDU Guard to block anyone from plugging in a rogue switch.
Use switchport port-security maximum 1 and switchport port-security violation restrict under each access interface. Enable BPDU Guard with spanning-tree bpduguard enable on the interface level.
Platinum
VLAN Access Map
Even with VLANs, a misconfigured router can re-introduce the problem. Implement a VLAN Access Map (VACL) on SW1 that explicitly denies any routed traffic from VLAN 20 reaching VLAN 10 — a defense-in-depth layer.
Research vlan access-map and vlan filter commands. You'll need a matching IP ACL. Note: this is a Catalyst feature — check whether your GNS3 image supports it before spending time on it.
